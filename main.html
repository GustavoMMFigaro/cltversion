Atividade Avaliativa - Luiz

Controle de versão:

O controle de versão é uma ferramenta para revisar o código a fim de encontrar alterações nele, ajudando no gerenciamento nas modificações do código do software. Dessa forma a equipe pode trabalhar no desenvolvimento de forma mais eficiente e rápida. O controle de versão guarda as versões antigas do código, possibilitando a comparação com versões anteriores caso ocorra algum erro no desenvolvimento, ajudando a correção do erro.

Controle de mudanças:

A utilização do processo de controle de mudanças é importante para gerenciar um progeto de desenvolvimento de software. Dito isso, controle de mudanças é uma forma para gestores análizarem o projeto, por meio de solicitações para os integrantes, a utilização desse processo visa a coordenação na equipe de gerenciamento de projeto, além de ter menos possibilidades de não seguir o cronograma.

Integração contínua:

A Integração contínua é um processo, onde novos códigos e alterações são adicionadas em repositórios do projeto de software, tendo em vista localizar e corrigir bugs no código. Esse processo ajuda na simplificação do desenvolvimento, gerando maior qualidade e permitindo umma entrega mais previsível.
